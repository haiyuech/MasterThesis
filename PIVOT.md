# PIVOT: Physics Informed and Vision Based Online Tuning

## 一句话总结

PIVOT 是一个面向自动驾驶赛车的动力学建模框架，用 DINOv2 视觉特征驱动参数网络，为物理动力学模型（或纯数据驱动模型）在线生成参数，实现对不同路面和工况的自适应，同时保持实时控制的计算效率。

---

## 解决什么问题

自动驾驶赛车中，MPC 控制器需要一个准确的车辆动力学模型来做轨迹预测和规划。核心矛盾在于：

- **固定参数的物理模型**（如自行车模型）：可解释、计算快，但参数是离线标定的，换个路面或轮胎磨损后就不准了。
- **纯数据驱动模型**（MLP、LSTM、TCN）：表达能力强，但在有限的真实数据上容易过拟合，且泛化性差。
- **视觉信息的利用**：摄像头能看到路面纹理、坡度等影响动力学的因素，但如何高效地把高维视觉特征融入控制回路是个挑战。

PIVOT 的目标是找到物理先验和数据驱动之间的最优平衡点，并通过系统实验给出明确答案。

---

## 整体架构

```
GoPro 图像 (854×480, 30Hz)
        │
        ▼
  DINOv2 特征提取 (384维语义特征 + 3维RGB统计 = 387维)
        │                         ← 异步运行，不在控制回路关键路径上
        ▼
  参数网络 ParameterMLP (~858K 参数)
        │
        ▼
  ┌─────────────────────────────────────────────┐
  │  输出的参数喂给以下其中一个动力学模型：       │
  │                                              │
  │  灰盒（物理信息）：                           │
  │    • LST: 17个参数 → 9个物理参数 + 8个残差修正  │
  │    • LTVL: 40个参数 → 时变线性模型的A/B矩阵    │
  │                                              │
  │  黑盒（纯数据驱动，超网络架构）：              │
  │    • MLP: 838个参数 → 小MLP的权重              │
  │    • LSTM: 19,141个参数 → 小LSTM的权重+状态    │
  │    • TCN: 821个参数 → 小TCN的卷积核+权重       │
  └─────────────────────────────────────────────┘
        │
        ▼
  RK4 积分器 → 9步轨迹预测 (Δt = 0.05s)
        │
        ▼
  MPPI / SIT-LMPC 控制器 → 车辆执行
```

### 核心设计思想

参数网络只运行**一次**（每个控制周期），生成一组紧凑的参数向量。之后 MPPI 采样 B=512 条候选轨迹时，每条轨迹的 9 步 rollout 都复用这组参数。这比"每步都过一遍大网络"快 1.5-2 倍。

## 七个模型的对比

### 固定参数基线（无学习）

| 模型 | 说明 |
|------|------|
| **DST** (Default Single-Track) | 非线性自行车模型，7维状态，离线标定参数，低速/高速切换机制 |
| **DTVL** (Default Time-Varying Linear) | 线性化动力学，5维状态，固定A/B矩阵 |

### 灰盒模型（物理+学习，PIVOT 的核心贡献）

**LST (Learned Single-Track)** — 网络输出 **17 个参数**，分两部分：

| 类别 | 参数 | 数量 | 作用 |
|------|------|------|------|
| 物理属性 | `m, I, lf, lr, h, C_Sf, C_Sr, mu, length` | 9 | 直接替换自行车模型中的物理参数（质量、转动惯量、前后轴距、重心高度、前后轮胎侧偏刚度、摩擦系数、车长） |
| 残差修正 | `alpha_delta, beta_delta, alpha_v, beta_v, alpha_yaw_rate, beta_yaw_rate, alpha_slip_angle, beta_slip_angle` | 8 | 对物理方程的 4 个输出通道（转向速率、纵向加速度、横摆加速度、侧偏角速率）做乘性(α)和加性(β)修正 |

LST 同时做了两件事：(1) 根据视觉特征推断当前工况下的物理参数（比如草地湿滑时降低摩擦系数 μ、调整轮胎刚度 C_Sf/C_Sr），(2) 用 α/β 补偿物理方程本身无法捕捉的建模误差。物理方程承担了主要的动力学预测，网络只负责"校准"和"补残差"。

**LTVL (Learned Time-Varying Linear)** — 网络输出 **40 个参数**（A 矩阵 5×7 = 35 + B 向量 5 = 5），生成状态相关的线性动力学矩阵，替代固定矩阵。

### 黑盒模型（纯数据驱动，超网络架构）

| 模型 | 网络输出 | 小网络结构 |
|------|---------|-----------|
| **MLP** | 838个权重 | 单隐藏层(64 units, tanh)，无记忆 |
| **LSTM** | 19,141个权重 | 单层LSTM cell(64 hidden units)，有时序记忆 |
| **TCN** | 821个权重 | 2层因果卷积 + 4步滑动窗口 |

**所有五个学习模型共享同一个 ParameterMLP 骨干网络**，区别仅在输出层维度和下游动力学结构。

## 关键实验结果

### 离线轨迹预测（9步 horizon）

- **LST 在所有模型中最优**：test pose loss = 0.57，比最好的黑盒模型 TCN (1.24) 好 2.2 倍
- **泛化能力**：LST 的 test/train ratio = 1.6，黑盒模型为 1.9-2.3（过拟合更严重）
- **非线性 > 线性物理先验**：LST vs LTVL = 0.57 vs 1.86，差 69%
- **视觉特征对物理模型不关键**：LST 去掉图像特征后 pose loss 从 0.57 变为 0.51（反而更好），而黑盒模型退化 1.4-2.4 倍

### 为什么 LST 赢？

根本原因是**输出空间维度**：

| 模型 | 参数网络输出层参数量 | 动力学参数数 |
|------|-------------------|------------|
| LST | ~2K | 17（9 物理 + 8 修正） |
| LSTM | ~4.9M | 19,141 |

LST 只需要学 17 个参数：9 个有物理含义的属性（质量、轮胎刚度等）+ 8 个残差修正。非线性自行车模型承担了绝大部分动力学预测，网络只做校准。LSTM 要学近 2 万个权重去从头拟合动力学，在有限真实数据上不可避免地过拟合。物理结构起到了**正则化**的作用。

### 计算效率

在 NVIDIA RTX A6000 上，MPPI B=512 时：

- LST 完整控制回路（参数网络 + rollout）：~573 μs
- 50Hz 控制频率（20ms 预算）下，仅占 **<3%**

## 论文贡献

1. **系统比较框架**：7 个动力学模型 × 3 种类别，同一数据集训练，控制变量地比较架构差异的影响
2. **PIVOT 灰盒架构**：LST 和 LTVL 模型，保留物理结构的同时通过视觉特征实现自适应参数估计
3. **两阶段评估**：离线轨迹预测 + 闭环赛车控制（SIT-LMPC），揭示预测精度和控制性能之间的关系
4. **真车验证**：非仿真实验，面对真实噪声、传感器延迟和地形变化
5. **实用洞察**：物理先验是最强的正则化手段；17 个参数的 LST 胜过 19,141 个参数的 LSTM
